## 🕹️ Hi I'm jjun, Software QA Engineer. 👏🏻

> QA는 단순히 버그를 찾는 역할이 아니라, 사용자가 경험하게 될 제품 퀄리티의 마지막 관문이라고 생각합니다. 

1. 안정적인 품질 보증을 위해 사용자 관점에서 앱과 웹 서비스를 바라보며 QA를 수행합니다.<br>
2. 기능의 정상·예외·경계 케이스를 고려한 TS와 TC를 체계적으로 작성하려고 합니다. <br>
3. 명확한 재현 절차와 문서를 기반으로 개발자와의 커뮤니케이션에 집중합니다.<br>
4. 문제점을 발견하고 원인을 분석하여 결함을 해결함으로써 제품의 완성도와 신뢰도를 높이는 과정에 가치를 둡니다.<br>
5. 리스크 중심적 사고를 통한 우선 순위를 판단하며, 지속적인 학습을 통해 더 안정적인 품질 보증을 지향합니다.<br>
